# scree_plot

def scree_plot(pca_object, export = False):
    # building a scree plot

    # setting plot size
    fig, ax = plt.subplots(figsize=(10, 8))
    features = range(pca_object.n_components_)

    # developing a scree plot
    plt.plot(features,
             pca_object.explained_variance_ratio_,
             linewidth = 2,             # sets the width of the lines
             marker = 'o',              # sets the style of the markers
             markersize = 10,           # sets the size of the markers
             markeredgecolor = 'black', # sets the color of the marker borders
             markerfacecolor = 'grey')  # sets the fill color of the markers

    # setting more plot options
    plt.title('Scree Plot')             # sets the title of the plot
    plt.xlabel('PCA feature')           # sets the label for the x-axis
    plt.ylabel('Explained Variance')    # sets the label for the y-axis
    
    # Example cutoff at the 4th component (which is index 3)
    plt.axvline(x=3, linestyle=':', color='red') # adds a vertical dotted line at x=3 with red color

    plt.xticks(features) # Setting the ticks for the x-axis based on the number of features

    # forcing the plot to display the vertical line
    plt.draw()

    # displaying the plot
    plt.show()

def scaler(df):
    """
    Standardizes a dataset (mean = 0, variance = 1). Returns a new DataFrame.
    Requires sklearn.preprocessing.StandardScaler()
    
    PARAMETERS
    ----------
    df     | DataFrame to be used for scaling
    """

    # INSTANTIATING a StandardScaler() object
    scaler = StandardScaler(copy = True)

    # FITTING the scaler with the data
    scaler.fit(df)

    # TRANSFORMING our data after fit
    x_scaled = scaler.transform(df)

    # converting scaled data into a DataFrame
    new_df = pd.DataFrame(x_scaled)

    # reattaching column names
    new_df.columns = list(df.columns)
    
    return new_df

def visual_cm(true_y, pred_y, labels = None):
    """
Creates a visualization of a confusion matrix.

PARAMETERS
----------
true_y : true values for the response variable
pred_y : predicted values for the response variable
labels : , default None
    """
    # visualizing the confusion matrix

    # setting labels
    lbls = labels

    # declaring a confusion matrix object
    cm = confusion_matrix(y_true = true_y,
                          y_pred = pred_y)

    # heatmap
    sns.heatmap(cm,
                annot       = True,    # Annotate the heatmap with the actual data values
                xticklabels = lbls,    # Sets the labels for the x-axis ticks (predicted labels)
                yticklabels = lbls,    # Sets the labels for the y-axis ticks (true labels)
                cmap        = 'Blues', # Sets the color scheme of the heatmap
                fmt         = 'g')     # Sets the string formatting for the annotated data

    plt.xlabel('Predicted')                         # Label for the x-axis
    plt.ylabel('Actual')                            # Label for the y-axis
    plt.title('Confusion Matrix of the Classifier') # Title for the heatmap
    plt.show()                                      # Displays the heatmap

This scree plot displays the variance explained by each principal component in the PCA analysis. The y-axis shows the variance explained, and the x-axis lists the principal components. The first point (PC0) is the highest, explaining the most variance. The second point (PC1) shows a significant drop but still accounts for a good amount of variance. After PC2, marked by the dotted red line, there's an 'elbow' in the plot, where the line begins to flatten out, indicating that additional components explain progressively less variance. This leveling off suggests that PC0 through PC2 are the most informative for understanding the structure in the data, and further components contribute less to the total variance explained.

The first dot (PC 0: 0.45)¶

For PC0, which represents the first principal component, its high value of approximately 45% indicates that it explains a significant portion of the variance in the dataset, suggesting that it captures the most prominent patterns or trends within the data. This makes PC0 crucial for understanding the fundamental structure of the dataset and identifying its primary underlying features. Additionally, PC0's dominance in variance explained implies that it may correspond to a key dimension along which the data points vary the most. Therefore, analyzing the relationship between variables and PC0 can offer valuable insights into the primary factors driving variation in the dataset.

The second dot (PC 1: 0.125)¶

Moving to PC1, which accounts for around 12.5% of the variance, it represents a substantial but comparatively smaller portion of the dataset's variability. Despite this, PC1 still captures meaningful patterns that differ from those captured by PC0, providing additional insights into the dataset's structure. By examining the loading scores of variables on PC1, one can identify the specific aspects of the data that contribute most to this component's variance. This analysis can help uncover secondary patterns or trends that are distinct from those identified by PC0, enriching the overall understanding of the dataset.

The third dot (PC 2: 0.12)¶

Finally, PC2, with an explained variance of approximately 12%, contributes further to understanding the dataset's structure by capturing additional patterns not fully represented by PC0 and PC1. While PC2 explains less variance than PC0 and PC1, its insights are still valuable for gaining a comprehensive understanding of the dataset's underlying characteristics. By exploring the relationship between variables and PC2, I can uncover unique features or relationships that contribute to the dataset's information.
